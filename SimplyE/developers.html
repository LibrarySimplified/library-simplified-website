Android and iOS Mobile Apps

The Android and iOS mobile apps share common archtecture. Both Apps are an OPDS Reader and an EPUB Player. Each uses its respective platfrom framework but do use some common cross platfrom components where possible.

OPDS Reader

Both app use platfrom specific framweworks and laguages for the OPDS reader and related views.

  • Android: Java and Android framework
  • iOS: Objective C, Swift and iOS framework
  • License: Apache 2.0

EPUB Rendering Engine

Both apps use custom implementation of the Readium SDK.

  • Languages: C++, JavaScript, Objective C and Java
  • License: BSD

Server Applications and Middleware

Repositories and resource sites

Core: https://github.com/NYPL-Simplified/server_core

The server applications are essentially organized into 3 areas - Core, Circulation Manager, Open Access Content Server and Metadata Wrangler.

  • Language: Python
  • Framework: Flask
  • Database: PostGresSQL
  • Repository and Issue Tracking: GitHub

Core

The Server Core contains functionality common between various SimplyE servers, including database models and essential class constants, OPDS parsers, and certain configuration details. The OA Content Server, Metadata Wrangler, and Circulation Manager all depend on this code base.

Circulation Manager

The Circulation Manager is the main connection between a library's collection and various client-side applications. It handles user authentication, combines licensed works with open access content from the OA Content Server, and data from the Metadata Wrangler.

Metadata Wrangler

The Metadata Wrangler utilizes and intelligently amalgamates a wide variety of information sources for library ebooks and incorporates them into the reading experience for users by improving selection, search, and recommendations.

Open Access Content Server

The Open Access Content Server collects and parses sources and preserves metadata for open access works, serving them up in a feed with verbose OPDS entries.
